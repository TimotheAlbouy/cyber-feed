Contrat d'interface de l'API Cyber-feed

Les routes qui nécessitent un jeton nécessitent qu'on le place dans l'en-tête Authorization.

Méthode Route Paramètres Jeton Réponse Description
POST /api/register.php

username: Nombre de caractères compris entre 3 et 36

password:

NON { token: string } Inscription et récupération du jeton d'accès
POST /api/login.php Saisies valides et correctes NON { token: string } Connexion et récupération du jeton d'accès
POST /api/add_feed.php Utilisateurs identifié - URL correct et complet OUI / Ajout d'un flux
DELETE /api/delete_feed.php Utilisateurs identifié - id du feed à supprimer OUI / Suppression d'un flux
GET /api/get_feeds_content.php Utilisateurs identifié OUI { feeds: [ { title: string, description: string, pubDate: string, imageUrl: string, imageType: string }, ... ] } Permet de récupérer le contenu des feeds
GET /api/get_feeds_url.php Utilisateurs identifié OUI { feeds: [ { id: string, url: string }, ... ] } Récupération de l'URL des flux de l'utilisateur
